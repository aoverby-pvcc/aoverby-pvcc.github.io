OPERATING SYSTEMS TERMINOLOGY
Term Definition
operating system The software that controls the overall operation of a computer
job queue The "waiting line" where jobs residing in mass storage wait to be executed
FIFO Stands for "first-in, first-out", meaning that the job first in the job queue is the first to be executed, the second in the queue is the second to be executed, so on, and so on.
interactive processing An operating system feature in which a program being executed can carry on a dialogue with a user through remote terminals
time-sharing An operating system feature in which services are provided to multiple users at the same time.
multiprogramming Multiprogramming is a time-sharing technique in which time is divided into intervals where the execution of a job can only be done one interval at a time. The intervals are then switched between rapidly to create the illusion of several jobs executing simultaneously.
application software The programs for performing tasks particular to the machine's particular use case. Application software I use: Excel, Tableau, VS Code Studio, Google Chrome.
system software Software which performs tasks which control the computer in general. Utility Managers, Settings, Printer software.
utility software A collection of system software which perform tasks fundamental to the system but not included in the operating system. Partitioning hard drives, formatting disks, moving files from USB sticks to local memory, etc.
graphical user interface (GUI) The graphical representation of files and programs through icons
shell Software which sends messages to and from the computer in the form of text commands and textual outputs. Command line, PowerShell, etc.
kernel The internal part of an operating system which stores the software performing basic functions. File managers, user files, drivers, memory manager, etc.
device driver Software units which communicate with peripheral devices (printers, USB drives, speakers, etc.) to control their function.
file manager Part of the kernal which maintains records of all files stored in the different mass storage locations (USB drives, hard drives, cloud-connected storage, etc.). Keeps up with where the files are, who can access those files, and which parts of mass storage are available for storage.
memory manager Part of the kernal which is tasked with maintaining the computer's use of main memory.
virtual memory The "fictional" memory space created by the technique of paging, or rotating the memory needed for programs and data back and forth between main memory and mass storage.
boot strapping How the operating system starts up. Often shortened to "booting".
ROM (Read-Only Memory) A portion of the main memory which can be read but cannot be altered. Its lack of volatility is useful for putting the operating system in a reliable location so a CPU can locate it and boot.
deadlock A resource allocation problem in which two or more processes are blocked from progressing because each program is waiting for a resource that is allocated to the other.
spooling Holding data for output at a later, more convenient time.
super user (or administrator) A user with highly privileged access to the operating system. Can edit programs inside of the kernal, is able to make user accounts, establish permissions to access certain files, etc.
multi-factor authentication User authentication method where access is granted only after presenting two or more pieces of evidence of the authentication methods of knowledge, possession, and inherence. For example, to log into a website you must enter a code sent to your cell phone to gain access.
