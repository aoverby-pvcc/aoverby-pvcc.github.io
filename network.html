Chapter 4 Terms:

Network Classifications These terms deal with the distance that information is travelling
  • Local Area Network (LAN) - connects devices located in a single location (university computer networks)
  • Metropolitan Area Network (MAN) - connects devices over an intermediate distances (a local community)
  • Wide Area Network (WAN) - connects devices over large distances (potentially across the world)
Network Models These terms deal with the topology of the network, or the pattern in which devices are connected
  • bus network: machines are all connected to a common communication line called a "bus"
  • start network: one machine serves as a central focal point to which all other machines are connected
Protocols Rules by which activities over networks are conducted
  • Carrier Sense, Multiple Access with Collision Detection (CSMA/CD) Ethernet:
    • Based on Ethernet standards on a bus network model. Does not work on a star network model
    • each messages is broadcast to all machines on the bus
    • Each machine monitors all messages but only keeps those addressed to itself
    • To transmit a message, machines wait until the bus is silent
    • If another machine also begins to trasmit a message, both machines will wait for a brief and independently random period of time before trying to transmit again
  • Carrier Sense, Multiple Access with Collision Avoidance (CSMA/CA) Wifi
    • Used by wireless networks -- commonly called WiFi
    • Each machine waits for a short period of time once it detects that the channel is silent before trying to transmit a message
    • If a busy channel is experienced during the short wait time, the machine waits for a randomly determined period of time before trying again
    • Machines that have been waiting are given preference for using the channel to transmit messages, so collisions with "newcomers" are avoided
Combining Networks Connecting different networks together to make extended communication systems
  • Connecting networks to make a larger network of the same type:
    • repeater:
      • a device that passes signals back and forth between two original buses (usually with some form of amplification
    • bridge:
      • a device that connects two buses, but does not necessarily pass all messages across the connection. It looks at the destination address and forwards a message across the connection only when that message is intended for a machine on the other side
    • switch:
      • a bridge with several connection points allowing it to connect several buses together rather than just two
  • Connecting networks that have incompatible characteristics. Building a network of networks:
    • router:
      • special purpose computers used to forward messages across disparate types of networks
    • gateway:
      • a loose term for a variety of things which all act as the "point" where the machine is linked to an internet
Client/Server Model
  • Client/Server Model: A convention for interprocess communication in which there are two basic roles: clients, which mak requests of other processes, and servers, which satisfy requests made by clients
  • Other Methods of Interprocess Communication:
    • Peer-to-Peer (P2P): machines can act as both clients and servers
    • distributed system: a network in which software units execute as processes on different computers
    • cloud computing: where large pools of shared computers on a network can be allocated for use by clients as needed
Internet Addressing The system of networks operated by Internet Service Providers (ISPs) can be classified in a hierarchy according to the role they place in the overall Internet structure
  • Tier 1: very high-speed, high-capacity, international WANs. Relatively few of these networks in the world. The "backbone" of the internet. Operated by large communication companies
  • Tier 2: more regional than tier-1 and less potent in capability, but still large networks operated by large communication companies
  • Tier 3: "access ISPs" operated by organizations which connect individuals to the Internet
IP address A unique identifying address for each computer in a network, known on the Internet as "Internet Protocol (IP) Addresses"
  • Internet Corporation for Assigned Names and Numbers (ICANN): a non-profit corporation established to coordinate the Internet's operation. It gives blocks of consecutively numbered IP addresses to different ISPs
  • Domain Name System (DNS): a system of using words instead of numbers to assign a unique address to each machine. These domain names are 1-to-1 matched with their IP addresses through numerous servers which contain the matches -- these are called name servers. The whole of these name servers is called the Domain Name System
  • Top-level domain (TLD): The suffix at the end of a domain name which relfect a website's classification (e.g., ".com" for "company", ".edu" for "educational", ".gov" for "governmental"
  • Country-code TLDs: top-level domains which indicate where in the world the website "belongs to" (e.g., ".de" for Germany, ".ca" for Canada)
Internet Protocols How different types of programs interact with machines over the Internet
  • File Transfer Protocol (FTP): network protocols for listing and copying files
  • Hyper Text Transfer Protocol (HTTP): network protocols for transmitting information and displaying that information via a webpage
  • Simple Mail Transfer Protocol (SMTP): network protocols for how emails are sent and received
World Wide Web
  • Uniform Resource Locator (URL): a unique address given to each document on the Internet. The URL contains the info needed by a browser to contact the proper server and request the specified document for display
  • Client-side activities: refers to actions performed by a user which request an action of some kind from a webserver
  • Server-side activities: refers to actions performed by a webserver as a result of a client-side request
Internet Four-Layer Software The software that controls network communication over the Internet
  • application layer:
    • prepares messages and provides destination address
    • receives messages
  • transport layer:
    • chops messages into packets
    • collects packets and resassembles message
  • network layer:
    • assigns intermediate address to each packet
    • detects that packet has reached its final destination
  • link layer:
    • transfers packet
    • receives packet
The Internet is:
  • packet-switching network
  • fault tolerant
  • redundant
Cybersecurity
  • malware: any software which is deemed "malicious"
  • virus: software which "infects" a computer by inserting itself into software that already exists. When that software is executed, the virus is also executed. Part of a virus' function is to replicate itself and "infect" other software and machines, other actions like degrading the operating system or fuctionality of the machine may also be a product of a virus
  • worm: an autonomous program that transfers itself through a network and forwarding copies of itself to other networks
  • Trojan horse: a program which enters a computer system disguised as a desirable program
  • spyware: software which collects information about activities on the computer on which it is installed and sends information back to another machine
  • phishing: a technique of obtaining information by explicitly asking for it
  • Distributed Denial of Service (DDoS) attack: a technique for stopping network activity by sending a machine a large amount of messages in a short period of time. This can be amplified by using more hosts, making it "distributed"
Protection
  • firewall: a program which filters traffic passing through a network
  • spam filter: a specific kind of firewall made to block unwanted email
  • proxy server: software which acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server
  • antivirus software: software which is used to detect and remove the presence of known viruses and other malware
  • cryptography: the study of sending and receiving messages securely in the presence of adversaries
  • public key encryption: an encryption technique in which two keys are used to send messages with one key being public and another key being private
